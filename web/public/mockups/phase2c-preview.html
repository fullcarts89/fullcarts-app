Item 9 · ✅ shipped

Skimpflation pipeline → published_changes

Migration 055 relaxes size_* / candidate_id NOT NULL constraints and adds skimp_score + nutrient_deltas columns to published_changes. New promote_skimpflation.py walks USDA cross-release diffs, matches each UPC to a pack_variant, and inserts one skimpflation row per qualifying entity. Idempotent — re-runs skip entities that already have one. Wired into the daily promote workflow.

What changes operationally:
• published_changes.change_type picks up a new value: 'skimpflation'
• New view skimpflation_events exposes only the skimpflation slice
• Existing event_evidence_summary view is rewritten to EXCLUDE skimpflation rows, so size-focused pages don't have to change
• Daily cron: nutrition_skimpflation findings are now first-class events alongside size shrinks
• Brand / product / dashboard counts include skimpflation in their "events" total
to deploy

Migration checklist

Run these once in the Supabase SQL Editor. Order matters — 055 must land before promote_skimpflation.py runs.

1. 054_product_index_view.sql — lights up /products
2. 055_published_changes_skimpflation.sql — relaxes NOT NULLs + adds skimp columns/view
3. 056_corporate_tree.sql — view; empty until Wikidata backfill runs
4. 057_google_trends_data.sql — table; empty until scraper runs
5. 058_consumer_reports_findings.sql — table; empty until scraper runs

The new scrapers (google_trends, consumer_reports, wikidata_manufacturer_backfill) are scheduled via GitHub Actions and will start populating on their cron without manual intervention.
